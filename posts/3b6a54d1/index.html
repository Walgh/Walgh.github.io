1 爬虫介绍

爬虫可以简单分为几步：抓取页面、分析页面和数据存储。在不同阶段有不同的python库以及方法供我们运用和实现。下面简单罗列各阶段常用的python库和工具。

  • 页面抓取：requests、Selenium和aiohttp等
  • 页面解析：正则表达式、lxml、Beautiful Soup、pyquery
  • 数据存储：MySQL、MongoDB

本文所述案例采用的技术思路为requests+正则表达式。由于requests库为第三方库，所以需要手动安装。虽然有很多第三方库可以解析网页，且正则表达式写起来相对繁琐，但采用正则表达式能够更直观且不需要对网页结构有很深认识。

3 爬虫实现

3.1 抓取主页

代码实现爬虫的第一步，是抓取首页数据。开始之前，需要确保已经安装好requests库。在进行模拟get请求时，需要设置请求头来伪装浏览器。具体的实现代码如下：

url = 'http://pic.netbian.com'
headers = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; WOW64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/70.0.3538.25 Safari/537.36 Core/1.70.3775.400 QQBrowser/10.6.4208.400'}
response = requests.get(url=url, headers=headers).text

运行并将response打印出来得到如下图结果：

可以看到请求得到的网页源码打印出来具有乱码，查看源码，可以发现红框所在处显示该网页采用gbk编码，因此在输出时需要对response进行gbk解码处理，具体实现语句为：

print(response.encode('ISO-8859-1').decode('gbk'))

此时输出就是正常无乱码字符了。

由之前对页面进行分析时可知，首页共有12个分类，此刻我们通过检查元素，找到其对应的源码，如下图红框所示代码。

此刻就要进入页面解析部分，本案例采用正则表达式进行信息提取。
正则表达式通过特定的语法，可以进行字符串匹配。它包含有许多匹配规则，常用的如下表1、表2所示。其中表1为常用匹配规则，表2为限定符，其具体将结合具体的实例进行说明。

为了进行信息提取，首先需要书写正确的正则表达式。对于上述红框部分代码，我们关心所有a字节信息，因此需要将所有a节点提取，此时需要用到通用匹配符.*（点星），其中点可以匹配任意字符（除换行符），星代表匹配前面的字符无限次，所以它们组合在一起就可以匹配任意字符了，提取目标，则需使用括号将想要提取的子字符串括起来，因此该段匹配的正则表达式书写可以为：

pattern = 'classify clearfix\">(.*)</div>'

有了正则表达式后，需要对字符串进行搜索，此处使用的是search()方法，它在匹配时会扫描整个字符串，然后返回第一个成功匹配的结果，具体代码如下所示，其中修饰符re.S表示匹配时也匹配换行符。

response = re.search(pattern, response, re.S)
print(response.group(1).encode('ISO-8859-1').decode('gbk'))

此时输出结果并不是我们想要的结果，这是因为匹配涉及一个贪婪匹配和非贪婪匹配的问题。在贪婪匹配下，.*会尽可能匹配更多的字符，.*后是</div>，但并没说是第几个，所以就会匹配到最后一个</div>，显然这样是不行的，这时只需要使用非贪婪匹配就好，非贪婪匹配的写法是.*?，多了一个?的效果具体看实例:

pattern = 'classify clearfix\">(.*?)</div>'
response = re.search(pattern, response, re.S)
print(response.group(1).encode('ISO-8859-1').decode('gbk'))

部分匹配结果为：

可以发现已经成功将各a节点源码正确提取出来，下面进行各分类的页面子链接以及分类名提取。由于每一个a节点代表一个分类，包含分类的子链接和分类名。提取单个分类的正则表达式可以写为：

pattern = re.compile('<a href=\"(.*?)\".*?>(.*?)</a>', re.S)

其中compile()方式是将正则字符串编译成正则表达式。
此时我们需要提取所有a节点的链接和分类名，search()方法已经不能满足需求，需要借助findall()方法，它会遍历字符串并返回所有满足正则表达式的内容，返回结果为列表类型，所以需要遍历一下来依次获取每组内容。

items = re.findall(pattern, response.group(1))
page_list_url = []
page_list_name = []
for item in items:
    page_list_url.append(item[0])
    page_list_name.append(item[1].encode('ISO-8859-1').decode('gbk'))
print(page_list_url)
print(page_list_name)

3.2 子页面抓取

3.1小节已经实现各分类链接和分类名的提取，本小节主要实现分类子URL的解析。首先需要进行URL拼接，以第一分类为例，在浏览器内点击风景分类得到该url为http://pic.netbian.com/4kfengjing/，为首页URL与3.1小节抓取的子链接拼接而成，因此可以据此构造完整子URL并发送get请求。

此刻拉到页面底端，可以查到页面总页数，通过查看源代码，可以用以下代码很方便地进行提取。

page_list_limit = re.search('\"slh".*?.html\">(.*?)</a>', page_list_text, re.S).group(1)
print(page_list_limit)

运行结果：

点击第二页查看URL变化为：http://pic.netbian.com/4kfengjing/index_2.html，第三页变化为http://pic.netbian.com/4kfengjing/index_3.html，因此该子页面链接规则为http://pic.netbian.com/4kfengjing/index_{d}.html， 在循环爬取时需要设置判断，代码如下：

if i == 1:  # 第1页爬取
    page_url = url + page_list_url[0]
else:  # 第2页及以上爬取
    page_url = url + page_list_url[0] + 'index_' + str(i) + '.html'

3.3 图片爬取

通过审查元素，可以发现详情页每一张图片信息对应一个li节点，节点内包含图片详情页面链接和图片源地址，但仔细审查就可发现该链接仅是缩略图，具体大图在详情页中包含，因此第一步需要将该页面下所有含照片的li节点提取出来。这里分两步实现，首先将ul节点提取出来，然后采用findall()方法进行各图片链接和名称的提取，具体代码如下：

page_list_text = requests.get(url=page_url, headers=headers).text
page_list_text = re.search('<ul class=\"clearfix\">(.*?)</ul>', page_list_text, re.S).group(1)
pattern = re.compile('<li>.*?<a href=\"(.*?)\".*?target=\"_blank">.*?<b>(.*?)</b>', re.S)
items = re.findall(pattern, page_list_text)
img_list_url = []
img_name = []
for item in items:
    img_list_url.append(item[0])
    img_name.append(item[1].encode('ISO-8859-1').decode('gbk'))
print(img_list_url)
print(img_name)

运行结果：

点击第一张图片查看该详情页url为http://pic.netbian.com/tupian/26426.html，可以发现为主页url和抓取的详情页url片段拼接而成，因此详情页拼接规则如下：

img_url = 'http://pic.netbian.com' + img_list_url[i]

对大图进行元素检查，可以找到其对应的代码段

采用正则表示式将其抓下来，并将url片段拼接成完整url。

img_page_text = requests.get(url=img_url, headers=headers).text
page_img_src = re.search('id=\"img\".*?<img src=\"(.*?)\" data-pic', img_page_text, re.S).group(1)
page_img_src = 'http://pic.netbian.com' + page_img_src

接下来就是最后一步，对图片进行存储：

img = requests.get(url=page_img_src, headers=headers).content
img_path = doc_name + '/' + name
with open(img_path, 'wb') as fp:
    fp.write(img)
    print(name + '下载成功!!!')

# coding=utf-8
# @brief 采用request请求和正则表达式进行彼岸图网图片抓取
# @author yishm

import requests
import re
import os


# 封装从栏目页进入内容页连接解析函数
def name(url, type_name, headers):
    url = url + page_list_url[int(type_name)]
    doc_name = './' + page_list_name[int(type_name)].encode('ISO-8859-1').decode('gbk')
    page_list_text = requests.get(url=url, headers=headers).text
    page_list_limit = re.search('\"slh".*?.html\">(.*?)</a>', page_list_text, re.S).group(1)
    while True:
        print("{}上限页数为{}页".format(page_list_name[int(type_name)].encode('ISO-8859-1').decode('gbk'), page_list_limit))
        page_need = input("请输入您要爬取{}的页数: ".format(page_list_name[int(type_name)].encode('ISO-8859-1').decode('gbk')))

        if page_need.isdigit() and 1 <= int(page_need) <= int(page_list_limit):  # 判断是否填写有误(包括页数判断)
            if not os.path.exists(doc_name):
                os.mkdir(doc_name)
            for i in range(1, int(page_need) + 1):
                if i == 1:  # 第1页爬取
                    spider(page_list_text, doc_name)
                else:  # 第2页及以上爬取
                    page_url = url + 'index_' + str(i) + '.html'
                    page_list_text = requests.get(url=page_url, headers=headers).text
                    spider(page_list_text, doc_name)
            break
        else:
            print("请重新输入正确的数字")


# 封装内容页图片连接解析函数
def spider(page_list_text, doc_name):
    page_list_text = re.search('<ul class=\"clearfix\">(.*?)</ul>', page_list_text, re.S).group(1)
    pattern = re.compile('<li>.*?<a href=\"(.*?)\".*?target=\"_blank">.*?<b>(.*?)</b>', re.S)
    items = re.findall(pattern, page_list_text)
    img_list_url = []
    img_name = []
    for item in items:
        img_list_url.append(item[0])
        img_name.append(item[1])

    for i in range(len(img_name)):
        img_url = 'http://pic.netbian.com' + img_list_url[i]
        name = img_name[i].encode('ISO-8859-1').decode('gbk') + '.jpg'
        img_page_text = requests.get(url=img_url, headers=headers).text
        page_img_src = re.search('id=\"img\".*?<img src=\"(.*?)\" data-pic', img_page_text, re.S).group(1)
        page_img_src = 'http://pic.netbian.com' + page_img_src
        img = requests.get(url=page_img_src, headers=headers).content
        img_path = doc_name + '/' + name
        with open(img_path, 'wb') as fp:
            fp.write(img)
            print(name + '下载成功!!!')
